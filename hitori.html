To play, click on a cell to switch it from light to dark. Once selected, arrow keys can be used to move around the canvas. Pressing the spacebar will toggle the value from on to off (white to black and reverse). In Error Display Mode (0), the default, only the existance of errors is indicated. In Mode (1), the number of errors are indicated, be they duplicate digits per row or column, adjacent blackened cells, or stranded "rivers" of white cells. In Mode (2), these errors are highlighted in colors. The canvas border will turn green if the grid is completed without error. You can click the "undo" buttom on the bottom of the page to undo previous moves.

The full set of rules for Hitori can be found online. The simple description includes a few basic rules.
  1. No single digit can exist "un-blackened" in a single row or column.
  2. There can be no blackened cells adjacent to another (diagonally adjacent is OK).
  3. The non-blackened cells must all form one contiguous "river" of cells, connected to each other in the vertical or horizontal direction.
On the "Game Play" tab, select puzzle 0 to see a completed sample grid. See the Game Play or Puzzles tab for how to invoke a demo.
This playground was written using javascript and is available open source. All code is checked into the github repo at ohmec/puzzles. Feel free to file bugs there!

Puzzles can be invoked by either a numerical entry from 0-0, or through a character description of the puzzle itself, entered into the Display form on the 'Game Play' tab. The numbered puzzles are hand-crafted of varying difficulty, roughly easy to hard. Puzzle 0 shows a completed correct puzzle. To describe a puzzle, the character string must be of the form: "WxH:<descriptor>", where W is a number indicating the width of the grid, and H the height. The descriptor is a definition of the grid contents from left to right, then top to bottom. A dot '.' can be used as a visual separator, for example between rows.
